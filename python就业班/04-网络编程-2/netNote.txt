mac地址：在两个设备之间通信时在变化，而ip地址在整个通信过程中不会发生任何变化

ip: 标记的是逻辑上的地址

mac: 标记的是实际转发数据的设备地址

netmask:和ip地址一起来确定网络号
默认网关：发送的 ip 不在同一个网段内，那么会把这个数据转发给默认网关


在浏览器输入 www.baidu.com 会经过一个怎样的过程?
    
    1.先要解析出 baidu.com 对应的 ip 地址
        1.先知道默认网关的 mac
            使用 arp 获取默认网关的 mac 地址
        2.组织数据发送给默认网关（ip 还是dns服务器的ip,但是 mac 地址是默认网关的 mac 地址）
        3.默认网关拥有转发数据的能力，把数据转发给路由器
        4.路由器根据自己的路由协议，来选择一个合适的较快的路径 转发数据给目的网关
        5.目的网关（dns服务器所在的网络），把数据转发给dns服务器
        6.dns服务器查询解析出的 baidu.com对应的 ip 地址，并发出它原路返回给 请求这个域名的 client
    2.得到了 baidu.com 的对应 ip 地址后，会发送 tcp 的三次握手，进行连接
    3.使用http协议发送请求数据发送给 web服务器
    4. web服务器收到请求之后，通过查询自己的服务器得到相应的结果，原路返回给浏览器
    5.浏览器收到数据后 通过自己的渲染功能来显示这个网页
    6.浏览器关闭 tcp连接，即 4次挥手
    7.完成整个过程
